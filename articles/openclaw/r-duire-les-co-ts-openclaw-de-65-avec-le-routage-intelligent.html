The OpenClaw Cost Optimization Playbook
Most people try OpenClaw and quit after seeing the bill.
I almost did too.
Three days in, I was burning $35-40 per day. That's over $1,000 a month just to run an AI agent. The math didn't work. I was ready to shut it down.
Then I discovered the settings that 90% of users miss.
Now I'm running the same workload for 65-70% less. Same output quality. Same speed. Completely different economics.
This is not about "using AI responsibly" or "being mindful of costs." This is about making OpenClaw financially viable so you can actually ship with it.
Here's everything I learned.

Why default settings destroy your wallet

OpenClaw sends everything to your primary model by default.
Heartbeat checks? Opus.
Calendar lookups? Opus.
Sub-agents doing parallel work? All Opus.
It's like hiring a neurosurgeon to check if you have a pulse. It works, but it makes zero financial sense.
The problem is not that OpenClaw is expensive. The problem is that the default configuration treats every single task like it requires maximum intelligence.
Most tasks don't.

The model tiering strategy that changes everything

Different models for different tasks. This is the foundation.
Complex reasoning → Opus or GPT-5.2. Worth it for hard problems. Strategic decisions. Creative work. Anything that requires deep thinking.
Daily work → Sonnet 4.6 or DeepSeek R1. 95% cheaper than Opus. Handles most of your actual workload without breaking a sweat.
Simple tasks → Gemini Flash. 98% cheaper than Opus. Perfect for lookups, parsing, classifications, anything mechanical.
Most people don't tier their models. They set Opus as the default and let it run wild. That's the entire problem.
The fix is simple. Route tasks based on complexity, not convenience.

The QMD skill that cuts research tokens by 90%

QMD is a local search tool instead of bloating your prompts with entire documents.
Here's how it works. Instead of sending entire docs to the API, QMD indexes your knowledge base locally. It uses BM25 plus vector search to find relevant snippets. Then it sends only those snippets to the model.
Built by someone at Shopify. Open source. Takes five minutes to install.
90% of my research tasks now use zero tokens.
Before QMD, every time OpenClaw needed to reference a doc, it would load the entire thing into context. Thousands of tokens per lookup. Hundreds of lookups per day.
After QMD, it pulls exactly what it needs. The rest stays local.
This one change cut my research costs by 90%.

Session memory is bleeding you dry

Every message loads around 50kb of history into context.
That's millions of wasted tokens per month if you're running multiple sessions daily.
The fix is one session initialization prompt. You tell OpenClaw to summarize context at the start of each session instead of loading the full history every single time.
Before: 50kb context per session.
After: 8kb context per session.
That's 40 cents per session down to 5 cents.
Add this to your system prompt and forget about it. It works in the background. No quality loss. Massive cost savings.

The "thinking" token trap nobody talks about

Models like o1 and DeepSeek R1 generate "thinking" tokens that you pay for but don't see.
These are internal reasoning steps the model runs before giving you an answer. They can be three to five times the visible output.
You're paying for all of it.
Use reasoning models only for complex reasoning. For everything else, use non-reasoning models.
This alone can cut costs by 60% if you're using o1 or R1 as your default.
I was running DeepSeek R1 for everything. Email sorting. Calendar checks. Simple classifications. Every single task was generating thousands of invisible thinking tokens.
Switching to Sonnet for routine work dropped my bill by more than half.

Use local models for repetitive work

Use Ollama with Llama 3.2 for tasks you do 100 times a day.
Email sorting. Calendar parsing. Simple classifications. Anything mechanical that doesn't require creativity.
One-time setup. Zero ongoing cost.
Reserve API calls for creative work and complex problem solving.
I route all my email triage through Ollama now. It runs locally. It's fast. It costs nothing.
That's hundreds of API calls per day that I'm no longer paying for.

Stop paying for web search

Perplexity API costs add up fast when you're running agents 24/7.
Exa AI is free web search for your bot. Setup takes 30 seconds via MCP. Your OpenClaw can now search the internet. Completely free.
This alone saved me $200 per month projected.
Before Exa, every time OpenClaw needed to look something up, it hit the Perplexity API. A few cents per call. Dozens of calls per day. It adds up.
Now it uses Exa. Same quality. Zero cost.

Let OpenRouter auto-route to cheaper models

80% of requests don't need Opus.
OpenRouter analyzes prompt complexity and routes automatically. Simple tasks go to sub-dollar models. Hard problems go to Opus.
Decision happens in under one millisecond. No quality loss.
You can also manually switch models mid-conversation if you want more control.
I let OpenRouter handle routing for most of my workflows now. It's smarter than I am at predicting which tasks need expensive models.
The result is that I'm only paying for Opus when I actually need it.

Heartbeats should cost almost nothing

OpenClaw sends periodic "are you alive?" checks to keep sessions active.
By default, these hit your paid API. Thousands of calls per month.
Route them to Gemini Flash instead. That's $0.10 per million tokens.
Or use local Ollama for zero cost. There are some bugs with the Ollama setup, but it works if you're willing to troubleshoot.
I'm using Gemini Flash for heartbeats now. The cost is basically zero. The savings are real.

Here's what you'll actually save

These are real numbers from optimized configs.
Light user (50 messages per day): $200 per month down to $70 per month.
Power user (200 messages per day): $943 per month down to $347 per month.
Heavy user (500+ messages per day): $3,000 per month down to $1,000 per month.
That's 65-70% cost reduction across the board. Same workload. Same quality. Completely different economics.

The mental model to remember

OpenClaw is not expensive. Default settings are expensive.
The platform is designed for flexibility, not cost optimization. That's fine. It means you have full control.
But you have to take control.
Tier your models. Install QMD. Fix session memory bloat. Avoid thinking token waste. Use Ollama for repetitive tasks. Route heartbeats to cheap models. Let OpenRouter auto-optimize.
Most people burn $500 to $2,000 per month on default settings. They try OpenClaw, see the bill, and quit.
That's leaving money and leverage on the table.
2026 is going to be unfair for builders who optimize early.
