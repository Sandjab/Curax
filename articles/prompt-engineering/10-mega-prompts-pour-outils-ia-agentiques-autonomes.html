Getting Started w/ AI Automation: 10 Essential Workflows (+ prompts)
For all of 2026, I've been testing autonomous AI tools non-stop.
OpenClaw, Manus, Cowork, Perplexity Computer - you name it.
Over time, I've built up a collection of mega-prompts that have genuinely changed how I work.
These aren't generic ChatGPT prompts you've seen recycled a hundred times; these are the exact prompts I use inside the most powerful agentic tools available right now.
These prompts actually get real work done without me babysitting every step.
10 prompts - each one designed to hand a real task off to an AI agent and come back to a finished result.

10. Perplexity Computer: Deep Financial Research Report

Prompt description: Conduct a full institutional-grade research report on any stock, crypto, or asset using live Perplexity Finance data - you can even use this to build a financial database (like a Bloomberg-style terminal) 
You are a senior financial analyst. I want you to conduct a full research report on [ASSET/TICKER].

Go to Perplexity Finance and gather the following data:

PRICE & MARKET DATA
- Current price, 24h change, 7d change, 30d change
- Market cap, volume, 52-week high/low
- Compare performance vs S&P 500 / BTC (depending on asset class)

FUNDAMENTALS (stocks only)
- Revenue, earnings, EPS, P/E ratio
- YoY growth rate
- Next earnings date
- Analyst consensus rating and average price target

NEWS & SENTIMENT
- Top 5 most recent news stories
- Overall sentiment: bullish / neutral / bearish
- Any major catalysts or upcoming events

INSIDER & INSTITUTIONAL ACTIVITY
- Recent insider buys or sells
- Major institutional holders and recent position changes

TECHNICAL PICTURE
- Is price above or below key moving averages (50d, 200d)?
- RSI reading — overbought, oversold, or neutral?
- Key support and resistance levels

Now compile everything into a structured report with these sections:

1. Snapshot — one paragraph summary of where this asset stands today
2. Bull Case — top 3 reasons to be long
3. Bear Case — top 3 risks to be aware of
4. Key Catalysts — upcoming events that could move price
5. Verdict — one sentence. What does the data say right now?

Format the output cleanly. Use headers. Be direct. No fluff.

9. Manus AI: Daily Automated Report

Prompt description: Manus connects to your Google Calendar and contacts, runs on a schedule every morning, and delivers a personalized daily brief, so you start every day knowing exactly what's ahead.
You are my personal chief of staff. Every morning at [TIME], 
run the following routine automatically and deliver my daily brief.

STEP 1 — PULL CALENDAR DATA
- Connect to my Google Calendar
- Pull all events for today
- Note: title, time, duration, attendees, location/link
- Flag any conflicts or back-to-back meetings with no buffer

STEP 2 — RESEARCH MY ATTENDEES
- For each person on today's calls, pull their LinkedIn or 
  contact info
- Surface: their role, company, last interaction, 
  any recent news about them or their company
- Flag anyone I haven't spoken to in 30+ days

STEP 3 — PRIORITY STACK
- Based on my calendar, rank today's top 3 priorities
- Identify any tasks or prep needed before each meeting
- Flag anything that looks underprepped or missing context

STEP 4 — COMPILE THE BRIEF
Structure the output exactly like this:

GOOD MORNING, [NAME]. HERE IS YOUR DAY.

DATE: [Day, Date]
WEATHER: [Location + conditions]

YOUR SCHEDULE
[Time] [Event title] with [Attendees]
[Time] [Event title] with [Attendees]
(repeat for all events)

PEOPLE INTEL
[Attendee name] — [Role, Company] — [One line context 
or recent news]

TODAY'S TOP 3 PRIORITIES
1. [Priority]
2. [Priority]
3. [Priority]

WATCH OUT FOR
[Any conflicts, missing prep, or flagged items]

ONE LINE TO START THE DAY
[A single motivating or focusing sentence]

Deliver this brief to [EMAIL or NOTIFICATION METHOD] 
at [TIME] every weekday.

8. OpenClaw: Build a Business Discord Server to Manage AI Sub-Agents

Prompt description: OpenClaw builds you a fully structured Discord server from scratch, sets up dedicated channels for each AI sub-agent, and creates a command center so you can run your entire AI operation from one place.
You are setting up a professional Discord server for me to use 
as my personal AI business command center.

The server should be structured so I can manage multiple AI 
sub-agents, track their outputs, assign tasks, and review 
results, all from one organized hub.

STEP 1 — CREATE THE SERVER STRUCTURE

Build the following categories and channels:

COMMAND CENTER
- #daily-brief (morning reports drop here)
- #task-queue (where I assign new tasks to agents)
- #priority-alerts (urgent flags only)

AGENT CHANNELS (one per agent)
- #agent-openclaw
- #agent-manus
- #agent-cowork
- #agent-perplexity

Each agent channel should have a pinned message with:
- What this agent is responsible for
- How to prompt it
- Expected output format

RESEARCH & OUTPUTS
- #research-reports
- #financial-briefs
- #content-drafts
- #competitor-intel

OPERATIONS
- #calendar-sync (morning briefs post here)
- #completed-tasks (archive of finished work)
- #errors-and-flags (anything that needs my attention)

PERSONAL
- #ideas-inbox (brain dump channel)
- #goals (pinned weekly priorities)

STEP 2 — SET UP ROLES

Create the following roles:
- Owner (me)
- Agent (assigned to each sub-agent bot)
- Reviewer (for any team members reviewing output)

STEP 3 — CONFIGURE PERMISSIONS

- Only Owner can post in #task-queue and #priority-alerts
- Agents can only post in their designated channels 
  and output channels
- All output channels are read-only for humans

STEP 4 — SET UP SUB-AGENT WORKFLOWS

For each agent channel, create a pinned workflow card:

AGENT: [Name]
TOOL: [OpenClaw / Manus / Cowork / Perplexity]
TRIGGER: [How to activate]
INPUT FORMAT: [What to send it]
OUTPUT: [What it delivers and where it posts]
FREQUENCY: [Scheduled or on-demand]

STEP 5 — LAUNCH CHECKLIST

Before finishing, confirm:
- All channels created and categorized
- All roles assigned with correct permissions
- Each agent channel has a pinned workflow card
- #daily-brief is connected to Manus morning brief output
- #financial-briefs is connected to Perplexity research output
- Server is clean, professional, and ready to operate

Deliver a summary of everything built when complete.

7. Cowork: Complete Desktop File Cleanse

Prompt description: Cowork cleans your entire desktop. Scanning for duplicate items, freeing up space & more *make sure to select folder*
I want you to do a full clean and reorganization of my 
computer. You have access to my file system. Go through 
everything and do the following:

STEP 1 — FULL SYSTEM SCAN
Navigate into every major directory:
- Desktop
- Downloads
- Documents
- Pictures
- Videos
- Applications
- Any other folders with significant file count

For each folder, catalog what's inside. Note file types, 
sizes, dates last modified, and anything that looks 
like clutter.

STEP 2 — IDENTIFY DUPLICATES
Scan across all folders for duplicate files.
Flag any file that appears more than once, even if 
stored in different locations or under slightly 
different names.
Do not delete anything yet. List them for my review.

STEP 3 — FLAG JUNK AND DEAD WEIGHT
Identify and list:
- Old installers and .dmg / .exe files
- Temp and cache files taking up space
- Empty folders
- Files that haven't been opened in 12+ months
- Screenshots and downloads that are clearly one-time use

STEP 4 — PROPOSE A FOLDER STRUCTURE
Based on what you find, recommend a clean folder 
structure that makes sense for how I actually work.
Show me the proposed structure before moving anything.

STEP 5 — EXECUTE THE REORGANIZATION
Once I approve, go into the folders and move files 
into the new structure.
Rename anything that has unclear or messy file names.
Keep the logic consistent throughout.

STEP 6 — CLEAR THE JUNK
Move confirmed junk to Trash.
Do not permanently delete anything without asking me first.
Show me a list of what you're moving to Trash before 
you do it.

6. Perplexity Computer: Live News Dashboard Builder

Prompt description: Perplexity Computer builds you a live, personalized news dashboard on any topic so you never miss what matters.
I want you to build me a live news dashboard using 
Perplexity to track everything happening around 
[TOPIC/INDUSTRY].

STEP 1 — DEFINE THE TOPIC MAP
Break [TOPIC] down into sub-categories worth tracking.

Example for AI:
- Model releases and research
- Company news (Anthropic, OpenAI, Google, Meta)
- Regulation and policy
- Business applications and tools
- Funding and acquisitions
- Public sentiment and viral moments

Confirm the sub-categories with me before proceeding.

STEP 2 — PULL LIVE DATA
For each sub-category, search Perplexity for:
- Top 3 stories from the last 24 hours
- Top 3 stories from the last 7 days
- Any breaking developments right now

For each story note:
- Headline
- Source
- Date published
- One sentence summary
- Why it matters

STEP 3 — IDENTIFY TRENDS
Across all the stories you pulled, identify:
- The 3 biggest themes dominating this week
- Any narratives that are building momentum
- Anything that seems underreported but significant
- Emerging names, companies, or technologies 
  showing up repeatedly

STEP 4 — BUILD THE DASHBOARD
Compile everything into a clean, structured dashboard:

TOPIC: [TOPIC]
LAST UPDATED: [Date and Time]

BREAKING NOW
[Story] — [Source] — [One line]

THIS WEEK'S TOP STORIES
[Sub-category]
1. [Headline] — [Source] — [Summary]
2. [Headline] — [Source] — [Summary]
3. [Headline] — [Source] — [Summary]
(repeat for each sub-category)

KEY TRENDS THIS WEEK
1. [Trend]
2. [Trend]
3. [Trend]

ONES TO WATCH
[Underreported story or emerging name worth tracking]

STEP 5 — SCHEDULE
Run this dashboard automatically every [morning/evening] 
and deliver it to [EMAIL or NOTIFICATION METHOD].

Flag anything that looks like a major breaking story 
immediately, outside the normal schedule.

5. Manus: Email Inbox Assistant

Prompt description: Turn Manus into a personal Email assistant - just connect your tools (Gmail) 
You are my personal email assistant. Every morning at 
[TIME], connect to my inbox and run the following routine.

STEP 1 — FULL INBOX SCAN
Connect to [Gmail / Outlook / other].
Pull every unread email from the last 24 hours.
Also surface any older emails still marked unread 
that have gone untouched for 48+ hours.

STEP 2 — CATEGORIZE EVERYTHING
Sort every email into one of these buckets:

URGENT — Needs my response today
IMPORTANT — Needs a response within 48 hours
FYI — No action needed, just worth reading
NEWSLETTER / CONTENT — Subscriptions and updates
JUNK — Spam, cold outreach, irrelevant noise

STEP 3 — DRAFT RESPONSES
For every email in URGENT and IMPORTANT:
- Read the full email thread for context
- Draft a response in my voice
- Keep it direct, professional, and concise
- Flag if you need more context from me before drafting

Present each draft like this:

FROM: [Sender]
SUBJECT: [Subject]
SUMMARY: [One line on what they want]
DRAFT RESPONSE:
[Your drafted reply]
SEND? Yes / Edit / Skip

STEP 4 — HANDLE THE NOISE
For NEWSLETTER / CONTENT emails:
- Summarize the top 3 most useful ones in one line each
- Archive the rest

For JUNK:
- Move to spam or trash automatically
- Flag any that look like they might be legitimate 
  before deleting

STEP 5 — INBOX REPORT
Once complete, deliver a summary:

INBOX REPORT — [Date]
Total unread: [#]
Urgent: [#]
Important: [#]
Drafts ready for review: [#]
Archived: [#]
Deleted: [#]

PENDING FROM YOU:
[List any emails where you need my input before 
drafting a response]

Deliver this report to [EMAIL / NOTIFICATION] 
at [TIME] every morning.
Ask before sending any email on my behalf.

4. OpenClaw: Expert vibe-coder (build anything)

Prompt description: Describe any app, tool, or product idea in plain English, and OpenClaw builds it for you. No coding knowledge required.
I want you to build [APP/TOOL/IDEA] for me from scratch.
I will describe what I want in plain English. You handle 
everything else.

STEP 1 — UNDERSTAND THE BRIEF
Here is what I want to build:

[DESCRIBE YOUR IDEA IN PLAIN ENGLISH]

Before writing a single line of code, confirm you 
understand the following:
- What this tool does
- Who it is for
- What the core features are
- What it should look like
- What tech stack makes the most sense

Ask me any clarifying questions before proceeding.

STEP 2 — PLAN THE BUILD
Break the build into clear phases:

PHASE 1: Core structure and layout
PHASE 2: Main functionality
PHASE 3: Design and UI polish
PHASE 4: Testing and bug fixes
PHASE 5: Final review and delivery

Show me the plan before you start building.
Get my approval on each phase before moving to the next.

STEP 3 — BUILD PHASE BY PHASE

For each phase:
- Show me what you are building before you build it
- Write clean, well-commented code
- Explain in plain English what each section does
- Flag any decisions you made and why
- Show me a preview or screenshot when possible

STEP 4 — HANDLE ERRORS
If you hit a bug or blocker:
- Tell me exactly what the issue is in plain English
- Propose two or three ways to fix it
- Pick the best one and proceed unless I say otherwise
- Never get stuck silently. Always flag and move forward.

STEP 5 — ITERATE
After each phase, ask me:
- Does this look right?
- Anything you want changed before we move on?
- Any features to add or remove?

Incorporate feedback immediately before moving to 
the next phase.

STEP 6 — FINAL DELIVERY
When the build is complete, deliver:
- The full working product
- A plain English summary of what was built
- Instructions for how to use it
- A list of possible next features to add in future

Build it clean. Build it fast. Explain everything 
as you go. I should be able to understand every 
decision you make without knowing how to code.

3. Cowork: Humanize Any Writing

Prompt description: Paste any AI-generated or robotic writing into Cowork, and it rewrites it to sound like a real person wrote it - natural, fluid, and impossible to detect.
I want you to take the following piece of writing and 
completely humanize it. Make it sound like a real person 
wrote it. Not an AI. Not a robot. A human with a voice, 
an opinion, and a personality.

HERE IS THE WRITING:
[PASTE TEXT HERE]

STEP 1 — DIAGNOSE THE PROBLEMS
Before rewriting, identify exactly what makes this 
writing sound artificial:

- Overly formal or stiff sentence structure
- Repetitive sentence length and rhythm
- Generic filler phrases and AI slop
- Lack of personality or point of view
- Unnatural transitions
- Words nobody actually says out loud
- Missing contractions, casual language, natural flow
- Any other red flags

List every issue before touching the text.

STEP 2 — DEFINE THE VOICE
Before rewriting, confirm the following:

TONE: [Casual / Professional / Conversational / Other]
AUDIENCE: [Who is reading this]
PLATFORM: [Where this will be published]
PERSONALITY: [How should the writer come across]
LENGTH: [Keep same length / Shorten / Expand]

Ask me if any of these are unclear.

STEP 3 — REWRITE THE PIECE
Now rewrite the full piece with these rules:

SENTENCE STRUCTURE
- Vary sentence length deliberately
- Mix short punchy sentences with longer ones
- Never write three sentences the same length in a row
- Start sentences with And, But, Or when it fits naturally

VOCABULARY
- Use words real people actually say
- Remove any word that sounds like it came from 
  a thesaurus
- Cut filler phrases completely:
  - "It is worth noting that"
  - "In conclusion"
  - "It is important to"
  - "Delve into"
  - "In today's world"
  - "Furthermore"
  - Any variation of these

PERSONALITY
- Add a point of view where appropriate
- Use contractions naturally
- Include at least one moment that feels personal 
  or specific
- Write like someone who actually cares about 
  the topic

FLOW
- Read every sentence out loud in your head
- If it sounds unnatural, rewrite it
- Transitions should feel effortless not mechanical

STEP 4 — SIDE BY SIDE COMPARISON
Present the output like this:

ORIGINAL:
[Original paragraph]

REWRITTEN:
[Humanized version]

Do this for each major section so I can see 
exactly what changed and why.

STEP 5 — SCORE THE OUTPUT
After rewriting, rate the final piece on:

HUMAN SCORE: [1-10]
NATURALNESS: [1-10]
PERSONALITY: [1-10]
AI DETECTION RISK: [Low / Medium / High]

If any score is below 8, identify what still needs 
work and fix it before delivering the final version.

STEP 6 — FINAL DELIVERY
Deliver the complete rewritten piece in full.
Clean. Ready to publish. Sounds like a human wrote it.

Probably the most valuable prompt here (reverse-prompting strategy). 
You are my **Workflow Interviewer**. Ask me these 30 questions to understand my work, then ask permission to build custom Computer workflows.

Start with these 5, then curate the next 25 based on my initial 5 answers. 

**ASK EXACTLY:**
1. "What's your #1 time sink this week?"
2. "Show me your most recent [tool/output]—what's broken about it?"
3. "What 3 repetitive tasks take 30+ min each?"
4. "Paste your best recent work—what made it successful?"
5. "What's your dream automated workflow?"

**AFTER MY FULL 30 ANSWERS:**

1. All Tools: Custom Skill Creator

Prompt description: Describe any repetitive task or workflow, and your AI agent builds a custom skill for it - so you never have to explain it again.
Works in Perplexity Computer, Manus, Claude, and/or OpenClaw! 
I want you to build a custom skill for me that I can 
reuse forever. I will describe the workflow I want 
automated. You turn it into a permanent, reusable skill.

STEP 1 — UNDERSTAND THE WORKFLOW
Here is the task I want to turn into a skill:

[DESCRIBE YOUR WORKFLOW IN PLAIN ENGLISH]

Before building anything, confirm you understand:
- What triggers this workflow
- What the input always looks like
- What the output should always look like
- What steps happen in between
- Any rules, preferences, or constraints I always apply

Ask me clarifying questions until you have the full picture.

STEP 2 — MAP THE WORKFLOW
Write out the full workflow as a step by step process:

SKILL NAME: [Name]
TRIGGER: [What kicks this off]
INPUT: [What I provide each time]
PROCESS:
  Step 1 — [Action]
  Step 2 — [Action]
  Step 3 — [Action]
  (continue for all steps)
OUTPUT: [What gets delivered]
FORMAT: [How the output is structured]
RULES:
  - [Any hard rules or preferences]
  - [Tone, style, length, format constraints]
  - [Things to always do]
  - [Things to never do]

Show me this map before building. Get my approval.

STEP 3 — BUILD THE SKILL
Write the full skill as a reusable prompt template.
It should be written so that any AI agent can pick 
it up and execute it perfectly every time.
Include placeholders like [INPUT] wherever I need 
to drop in new information each time I use it.

STEP 4 — TEST THE SKILL
Run the skill once using a real example I provide.
Show me the full output.
Ask me:
- Did this hit the mark?
- Anything to adjust in the workflow?
- Any steps missing or unnecessary?

STEP 5 — REFINE AND FINALIZE
Incorporate any feedback.
Rerun the skill with the same example to confirm 
the fix worked.
Repeat until the output is exactly what I want 
every single time.

STEP 6 — DELIVER THE FINAL SKILL
Package the skill as a clean, ready-to-use prompt 
I can save and deploy across any tool:
- OpenClaw
- Manus
- Cowork
- Perplexity Computer
- Claude

Deliver it in this format:

SKILL: [Name]
CREATED: [Date]
WORKS WITH: [Tools]
DESCRIPTION: [One line on what this skill does]
TRIGGER: [What to say to activate it]

FULL PROMPT:
[Complete reusable skill prompt]

USAGE NOTES:
[Any tips on getting the best results]
[Common variations or modifications]
